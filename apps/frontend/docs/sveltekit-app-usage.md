# SvelteKit 앱 사용 방법

## 준비
프로젝트 루트에서 실행한다.

```bash
cd "/mnt/c/Users/osc21/Downloads/cuchen 2/cuchen/apps/frontend"
npm install
```

Node가 PATH에 없으면 현재 환경 기준으로 아래처럼 실행한다.

```bash
PATH=/home/castle6116/.nvm/versions/node/v22.16.0/bin:$PATH npm install
```

## 백엔드 연결
기본 백엔드 주소는 아래 값이다.

```text
http://127.0.0.1:3000
```

다른 주소를 사용할 때는 `.env`에 `VITE_BACKEND_URL`을 설정한다.

```bash
cp .env.example .env
```

## 개발 서버 실행
SvelteKit 개발 서버를 실행한다.

```bash
npm run dev
```

브라우저에서 Vite가 출력한 주소로 접속한다. 기본 주소는 아래와 같다.

```text
http://127.0.0.1:5173/
```

포트가 사용 중이면 Vite가 다른 포트를 자동으로 안내한다.

## 다운로드
다운로드는 브라우저 기본 다운로드를 사용한다.

- 행 다운로드: 해당 제품번호의 이미지 묶음을 ZIP으로 저장
- 선택 다운로드: 선택된 제품들의 이미지 묶음을 ZIP으로 저장

## 화면 사용
1. 앱을 실행하면 제품 단위 파일 목록이 바로 표시된다.
2. 상단 필터에서 `시작일`, `종료일`, `제품번호`, `이미지 구분`, `검사 결과`를 선택한다.
3. `검색`을 누르면 조건이 적용되고 페이지는 1페이지로 돌아간다.
4. `초기화`를 누르면 모든 필터가 해제된다.
5. 테이블 하단에서 페이지 크기와 이전/다음 페이지를 조작한다.
6. 목록은 같은 제품번호의 이미지 4종을 하나의 행으로 묶어서 표시한다.
7. 테이블 왼쪽 체크박스로 제품을 선택하면 헤더에 선택된 제품 개수가 표시된다.
8. 헤더의 `선택 다운로드` 버튼을 누르면 선택된 제품의 이미지들이 하나의 ZIP으로 저장된다.
9. 각 행의 `상세` 버튼을 누르면 같은 제품번호의 이미지 4종과 metadata가 모달에 표시된다.
10. 각 행의 `다운로드` 버튼을 누르면 해당 제품의 이미지 묶음을 ZIP으로 저장한다.

## 검증 명령
변경 후 아래 순서로 확인한다.

```bash
npm test
npm run check
npm run build
```

빌드 결과는 `.svelte-kit`과 정적 배포용 `build/` 디렉터리에 생성된다.
