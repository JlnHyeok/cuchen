# Frontend Plan

## 1. 조회 경로 정리
- SvelteKit 프론트엔드는 `page + pageSize + filter` 쿼리 모델을 가져야 한다.
- 목록 조회는 backend HTTP API client의 `listFiles()` 또는 `search()`로 통일한다.
- 이미지 보기와 다운로드는 backend API를 경유한다.

## 2. 상태 모델 정리
- 선택된 버킷
- 현재 페이지와 페이지 크기
- 서버 전달용 필터 상태
- 선택된 파일
- 로딩/에러 상태

## 3. API 연결
- 목록 조회 API에 필터 파라미터를 넘긴다.
- 페이지 전환 시 이전 응답을 무시할 수 있어야 한다.
- 이미지 미리보기와 다운로드는 backend API만 사용한다.

## 4. UX 처리
- 로딩 중 이전 페이지 데이터가 남아 보이지 않게 한다.
- 빈 결과와 오류를 분리해서 표시한다.
- 대량 데이터에서는 클라이언트 전체 필터링을 하지 않는다.

## 5. 테스트
- 렌더링 테스트
- 페이지 전환 테스트
- 서버 필터 전달 테스트
- 이미지 조회 및 다운로드 테스트
