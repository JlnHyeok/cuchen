# Frontend Review Points

## 기능 리뷰
- SvelteKit 프론트엔드가 전체 데이터셋을 받아서 로컬 필터링하지 않는지 본다.
- 목록 조회가 backend API로만 나가고, 페이지와 필터가 함께 전달되는지 본다.
- 이미지 보기와 다운로드가 backend 경로에만 머무는지 본다.
- 상세 패널이나 모달이 목록 페이지와 충돌 없이 갱신되는지 본다.

## UX 리뷰
- 로딩 중 이전 데이터가 남아 보이지 않는지 본다.
- 빈 데이터와 로딩 상태가 구분되는지 본다.
- 대량 데이터에서도 페이지 전환과 필터 적용이 멈춘 것처럼 보이지 않는지 본다.

## 위험
- 목록을 클라이언트에서 다시 가공하면 MongoDB 전환 이점이 사라진다.
- 선택 상태와 서버 응답이 어긋나면 상세 표시가 쉽게 꼬인다.
- backend 장애 시 명시적인 오류 상태가 없으면 화면이 멈춘 것처럼 보인다.

## 관측 포인트
- 초기 렌더 시간
- 목록 전환 지연
- 필터 반응 시간
- 상세 패널 갱신 시간
- 다운로드 선택 상태 유지 여부
