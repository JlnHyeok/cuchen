# 문서 관리 명세

## 역할
문서와 구현의 불일치를 발견하고 고친다.

## 책임
- API 변경 시 문서 갱신
- 데이터 구조 변경 시 명세 갱신
- 리뷰 포인트 보강
- 오래된 예시 제거
- 새 에이전트 역할 추가 시 루트 `AGENTS.md`와 `docs/agent/` 문서 갱신
- 중복 문서 제거 또는 통합
- `docs` 폴더명이 문서 역할과 어긋나지 않게 유지

## 작업 기준
- 코드보다 문서가 우선이다.
- 구현이 바뀌었으면 spec과 review도 같이 바꾼다.
- 문서만 바뀌고 코드가 안 바뀌는 경우는 변경 이유를 남긴다.
- 문서 역할이 `참고`, `계약`, `계획`, `리뷰`, `배포`, `보관` 중 어디인지 분명해야 한다.
- 즉시 실행해야 하는 공통 지침은 루트 `AGENTS.md`에 두고, 세부 운영 규칙은 `docs/agent/`에 둔다.

## 관측 항목
- 문서-코드 불일치 건수
- stale section 수
- missing review point 수
- 역할이 겹치는 문서 수
