# 성능 측정 명세

## 목적
데이터가 늘어나도 시스템이 유지되는지 수치로 확인한다.

## 측정 축
- ingest 처리량
- 목록 조회 지연
- 필터 조회 지연
- 상세 조회 지연
- 이미지 보기 지연
- 다중 다운로드 지연

## 데이터 규모
- 1,000건
- 10,000건
- 25,000건

## 측정 원칙
- 같은 데이터셋으로 반복 측정한다.
- p50과 p95를 모두 본다.
- 측정 환경과 데이터 크기를 함께 기록한다.
- 결과는 수치와 로그 파일로 남긴다.

## 비교 대상
- MongoDB 기반 조회
- MinIO 기반 이미지 제공
- record 파일 기반 조회가 남아 있으면 참고 비교만 한다

## 병목 관측
- 인덱스 미사용
- 클라이언트 과다 렌더링
- 대용량 이미지 전송
- 페이지 전환 시 전체 재요청
- 설치 직후 초기 warm-up 지연
